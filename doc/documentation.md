# Documentation Guidelines

Documentation is part of the implementation work.

## Requirement

When implementing or changing a feature, write or update the relevant
documentation in the `doc/` folder.

## Documentation Should Cover

- The user-facing behavior or workflow.
- Required configuration, environment variables, or setup steps.
- Important architecture or integration decisions.
- Operational notes, verification steps, or known limitations.

## Placement

- Keep durable project documentation in the `doc/` folder.
- Use focused Markdown files with descriptive names.
- Link important documentation directly from `README.md` when it is needed for
  onboarding, setup, or normal development.
